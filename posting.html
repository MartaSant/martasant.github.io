Posting guide · Food Facts

A private step-by-step guide for Marta: how to publish a new article, update the blog and home page, add SEO meta tags, and send a newsletter.

Quick ritual (very short version)

  1. Write the post text (title, excerpt, date, categories, meta label, read time).
  2. Go to editor.html → generate:
    • postX.html
    • snippet for blog.html
    • snippet for index.html
  3. Create postX.html, paste full code.
  4. Paste the snippets into blog.html and index.html.
  5. Go to seo-helper.html → generate meta → paste into postX.html inside <head>.
  6. Go to newsletter-helper.html → generate email HTML → paste into MailerLite.
  7. Commit + push to GitHub → check the live pages.

1. Prepare the post content

Before touching any HTML, prepare the content in a text editor (Google Docs, Word, etc.).

  • Title – the main headline of the post.
  • Excerpt – 1–2 short sentences that summarise the post (used in cards and meta).
  • Date (displayed) – e.g. November 28, 2025.
  • Meta label – a small label like Intro, Overproduction, Label deep dive. If you leave it empty, the editor will use the first category.
  • Reading time (optional) – e.g. 12 min read.
  • Categories – choose among:
    • Production
    • Waste
    • Sustainability
    • Innovation
    • Consumers
    • Policy
    • Health
    • Opinions
  • Body – the full text. Separate paragraphs with a blank line.

2. Use editor.html to generate code

Open the internal editor page:

  • Visit editor.html in the browser (e.g. https://martasant.github.io/editor.html).

Fields to fill

  1. Post number – choose the next free number:
    • If you already have post0.html, post1.html, post2.html, the next one is 3.
    • This will create a new file called post3.html.
  2. Title – paste the post title.
  3. Meta label – e.g. Overproduction, Intro, Label deep dive (or leave empty and it will use the first category).
  4. Date – e.g. November 28, 2025.
  5. Estimated reading time – optional (e.g. 12 min read).
  6. Categories – tick all relevant categories. These control the filters in blog.html.
  7. Short excerpt – paste the 1–2 sentence summary.
  8. Full post content – paste the entire article.
    Use an empty line between paragraphs. Line breaks inside a paragraph will become <br /> tags.

Then click Generate code. The page will show:

  • Post HTML file – full content for postX.html.
  • Blog list snippet – <article class="post"> block for blog.html, with:
    • a comment like <!-- Post3 -->,
    • title (not a link),
    • meta line,
    • excerpt,
    • green button “Read the full post”.
  • Home “Latest posts” snippet – <article class="post-card"> for the home page:
    • comment <!-- Post3 -->,
    • title, meta, excerpt,
    • green button “Read the full post”.

3. Create postX.html

  1. In your project folder, create a new file:
    • Example: post3.html.
  2. Open the file and paste the entire “Post HTML file” code generated by editor.html.
  3. Save the file.

Tip: you can always open an existing post file (like post1.html) to compare and check that the structure feels similar.

4. Add the post to blog.html

  1. Open blog.html.
  2. Find the block: 
    <section class="content">
  ...
</section>
  3. Inside that section, place the new snippet from editor.html:
    • Paste it above or below the existing posts.
    • Keep posts ordered from newest (top) to oldest (bottom), if you want.

The categories you selected in the editor are already attached as data-categories, so the filters and search will work automatically.

5. Add the post to “Latest posts” on index.html

  1. Open index.html.
  2. Find the “Latest posts” section (it contains <article class="post-card"> blocks).
  3. Paste the “Home Latest posts” snippet from editor.html:
    • Example: <!-- Post3 --> followed by <article class="post-card">....
  4. If you want to keep the home page clean, you can limit this section to the 3–4 most recent posts:
    • When you add a new one, you can remove the oldest from this section (but it will remain on blog.html).

6. Add SEO & social meta with seo-helper.html

The goal: each postX.html should have a clean <title>, a meta description and Open Graph / Twitter tags so links look good when shared.

Steps

  1. Open seo-helper.html (e.g. https://martasant.github.io/seo-helper.html).
  2. Fill the fields:
    • Post number – e.g. 3 (this is only used for a comment like <!-- Post3 meta -->).
    • Page title – usually the same as the article title.
    • Short description – 1–2 sentences, can be similar to the excerpt.
    • Page path – e.g. post3.html.
    • Site base URL – usually https://martasant.github.io (one day this can become your custom domain).
    • Image (optional) – e.g. images/post3-cover.jpg or a full https:// URL.
    • Image alt text (optional) – a short description of the image.
  3. Click Generate meta tags.
  4. Copy the generated block.
  5. Open post3.html, find the <head> section and paste the meta tags right under: 
    <meta charset="UTF-8" />

7. Create the newsletter with newsletter-helper.html

This step prepares a nice HTML email in Food Facts style. You’ll paste it into MailerLite or another email platform.

Steps

  1. Open newsletter-helper.html (e.g. https://martasant.github.io/newsletter-helper.html).
  2. Fill the fields:
    • Email subject – subject line for the inbox.
    • Preheader – small line next to the subject (optional but useful).
    • Newsletter title (inside email) – big title at the top of the email.
    • Intro paragraph – a short intro.
    • Main content – body of the email.
      Use blank lines between paragraphs.
    • Optional bullet list – one bullet per line.
    • CTA button label – e.g. Read the full article.
    • CTA button URL – link to the live post, e.g. https://martasant.github.io/post3.html.
    • P.S. – closing note (optional).
  3. Click Generate email HTML.
  4. Copy the entire HTML from the output box.

Use it in MailerLite

  • Create a new campaign in MailerLite.
  • Set the same subject and preheader you used in the helper.
  • Choose an editor option that allows custom HTML (or an HTML block).
  • Paste the generated email HTML.
  • Replace the unsubscribe line with MailerLite’s official unsubscribe block if required.

8. Commit, push and check

  1. Make sure you have saved all updated files:
    • postX.html
    • blog.html
    • index.html
    • any new images
  2. In your Git tool or terminal:
    • git add .
    • git commit -m "Add postX: [short title]"
    • git push
  3. Wait a short moment for GitHub Pages to update.
  4. Check in the browser:
    • https://martasant.github.io/postX.html – the new article.
    • https://martasant.github.io/blog.html – appears in the list and filters.
    • https://martasant.github.io/ – appears in Latest posts if you added it.

Optional notes & reminders

  • Caching – if a page doesn’t look updated:
    • Use a “hard refresh” in the browser:
      Ctrl + F5 (Windows) or Cmd + Shift + R (Mac).
  • Categories logic – in blog.html, category filters use AND logic:
    • If you select Production + Policy, a post must have both categories to appear.
  • Private helper pages – pages like editor.html, seo-helper.html, newsletter-helper.html, posting.html:
    • are public if someone knows the URL,
    • but they are not linked in the main navigation, so they are effectively “hidden” for normal visitors.
